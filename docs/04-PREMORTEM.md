# DMEN — Premortem (pre-build risk autopsy)

_8 adversarial failure-lens autopsies (imagining a failed Dec-2026 launch) + synthesis. Ranked by severity × likelihood. These corrections are folded into the build._

## Top risks (ranked)

Dedupes 4 modes (Conversion, India, Architecture lenses). | Make a real ₹1 end-to-end purchase (UPI intent + COD-OTP + one card) on a physical mid-range Android over 4G the gate-zero milestone in week 1, behind a feature flag, BEFORE PDP editorial polish. ## Pre-build checklist (launch blockers / owner actions)

- [ ] Spin up a free Shopify dev store, seed the 9 real SKUs as an India market (INR, tax-inclusive display), enable inventory tracking, and confirm quantityAvailable + availableForSale actually flow through the Storefront API before any scarcity/low-stock UI is built.
- [ ] Run @shopify/graphql-codegen against the live Storefront schema and commit the generated types as THE commerce interface; the mock adapter must conform to them (not the reverse). Money is a {amount:string,currencyCode} object everywhere, lists normalized from edges/node at the adapter boundary, ids treated as opaque GIDs, handles sourced from the dev store so routes/breadcrumbs don't 404 at swap.
- [ ] Decide the checkout architecture NOW: hosted Shopify checkout via cart.checkoutUrl redirect (recommended) vs Hydrogen/checkout extensibility. Confirm Shopify Payments (or Razorpay/GoKwik) with UPI intent + COD + one wallet, and complete one real ₹1 end-to-end purchase on a physical mid-range Android over 4G as gate-zero.
- [ ] Resolve the open ops constants as launch blockers tied to real data: free-shipping threshold (modeled against single-SKU ₹699 price points), returns/exchange window (push to 10-15 days, define reverse-pickup, refund method/timeline, hygiene exclusions for worn gymwear), and COD policy (OTP confirmation, codFee, address scoring, RTO mitigation).
- [ ] Pick and wire a real India serviceability/ETA source (Shiprocket/Delhivery/GoKwik) behind a DeliveryEstimateProvider port before building the PDP pincode field; never hardcode a date.
- [ ] Lock the commissioned high-key grit photoshoot: budget, casting, studio, shot list, and a single shared grade LUT; record model height + chest/waist + exact size worn per setup into asset metadata. Back-schedule launch from shoot delivery.
- [ ] Export the bespoke DMEN wordmark from DMEN-Brand/Light theme.svg as an inline <BrandWordmark> SVG component (into the existing src/components/brand/Logo.tsx) and extract the red swoosh as a brand glyph token; ban rendering 'DMEN' in Anton via CI grep.
- [ ] Build a metric-adjusted Anton fallback (next/font size-adjust/ascent-override matched to Arial Narrow) and aggressively subset Anton to Latin + hero glyphs; decide font-display:optional for the hero H1.
- [ ] Define the granular analytics event schema and stand up a funnel dashboard in week 1, before component build.
- [ ] Pick the review platform (Judge.me/Okendo/custom) keyed to Shopify product GID/handle, and launch the pre-launch gifting program to source real seeded reviews + consented UGC photos.
- [ ] Pull real per-SKU tech-pack data: size charts (in+cm), GSM, blend, fit type, care — no shared default chart; bake modelHeightCm, modelSizeWorn, fitType as REQUIRED non-nullable fields so the caption/spec components cannot compile without real values.
- [ ] Commission/schedule a dedicated copy pass for 9 differentiated product narratives + 4 collection straps + ≥3 Journal posts + About arc, with a dated content-freeze and a no-clone similarity audit.
- [ ] Run a 5-second first-click test + open card-sort with 8-10 cold audience users on the nav labels (Apparel/The Drop/Journal) before locking them.
- [ ] Build the contrast matrix (axe-core/script) asserting every token-on-token pair actually used, tagged text (4.5:1) vs graphical (3:1); fix --color-warning #B26A00 (fails at 4.06:1) and verify the red hero word against a solid text plate, not the video frame.

## Build-order adjustment
Invert the sequence so the money path and the data contract lead, not trail. Week 1 is a vertical slice, not horizontal polish: stand up the Shopify dev store with the 9 real SKUs (India market, inventory tracking), codegen the types, build the async server-authoritative mock adapter + DeliveryEstimateProvider against those types, wire ONE SKU through real Shopify hosted checkout with UPI+COD, and complete a real ₹1 transaction on a physical mid-range Android over 4G as gate-zero. In parallel and equally critical-path, lock the photoshoot (budget/casting/LUT) and launch the review-gifting program, since both are long-lead and block launch independent of dev. Only after the transacting slice and the content pipeline are de-risked do you build out the PDP editorial stack, A+ modules, and home identity sections — with the granular analytics dashboard and CWV field-p75 CI gate live from the first deployed page so leaks and regressions are visible from soft-launch, not month six.

## Design improvements (beyond original spec)

- Build a bespoke type signature beyond Anton: cut a custom hero-only headline and a distinctive pipe-treated MEN | MINDSET | HUSTLE motif derived from the logo letterforms (the D-swoosh as the pipe/diamond), plus a tabular price style — so the store is recognizably DMEN, not the generic Anton-condensed-caps 2025 DTC template, and passes the 'distinguishable with logos removed' test.
- Turn the logo's red D-swoosh into a system-wide brand glyph: a ≤16px recurring signature mark that quietly anchors section corners, the marquee separator (◆→swoosh), bullet markers, and the order-confirmation, so the white canvas carries chromatic identity through the long red-less stretches without breaking action-red discipline.
- Make the 'Own the white canvas' bet legible through deliberate asymmetry and oversized negative space as a signature — one dominant element per section, off-center compositions, and an editorial baseline grid — codified as enforceable rules so the premium feel is engineered, not hoped for.
- Treat the model-height + size-worn caption as a designed fit-certainty device, not a footnote: a consistent on-image caption lockup ('6'1" · wears M · athletic fit') plus a fit-confidence micro-component near the size tiles, turning the cheapest return-reducer into a visible brand mechanism.
- Design an honest low-review-count state as a first-class premium component (e.g. 'Among the first to wear it — 7 verified Rising Men') so a launch catalog reads intentional and trustworthy instead of empty or faked, converting a constraint into a founders-era brand asset.
- Carry the physical unboxing into the digital flow as a recurring motif: the black mailer + white/red tissue + manifesto insert previewed in the PDP gallery and echoed on the confirmation screen ('Welcome to the Rising'), making a digital-first brand feel tactile and giving the post-purchase moment real craft.
- Replace the cosmetic endowed-progress bar with an honest, branded continuous-momentum indicator that matches the real Shopify redirect flow, and brand the hosted checkout (logo, red CTA, Anton headings) so the handoff feels like one continuous DMEN experience rather than a jarring third-party jump — protecting the quiet-premium trust positioning.
- Add a value-justification microline at the price block anchored on cost-per-wear / GSM durability ('240 GSM · holds shape through 100 washes · ~₹X per wear') plus a single genuine first-order/prepaid incentive in the one Offers slot, so the ₹699-1999 premium reads as earned conviction rather than arbitrary — building perceived value without a discount ladder that cheapens the brand.
- Define a primary-action hierarchy independent of color (size, weight, full-width bottom-anchored placement, surrounding whitespace) so the next step is unmistakable on action-dense cart/checkout screens even where red is rationed or repeated — making the funnel clearer AND more disciplined at once.
- Commission a cohesive single-LUT grade as a brand-defining art-direction spec (high-key, hard directional light, real sweat/weave) with per-breakpoint byte budgets that treat 'looks gritty at 40KB on a phone' as a sign-off criterion — so the anti-AI photography is both the strongest distinctiveness lever and fast on 4G, two goals usually in tension reconciled by design intent.
